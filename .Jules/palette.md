## 2025-02-20 - Accessible and Deep-linkable Tags
**Learning:** Tags dynamically generated in components (Astro and vanilla JS) must carefully handle slugs (like URI-encoding, lowercasing, and removing spaces) so that `#` anchor deep-linking works reliably. Also, generating long lists of tags inside `div`s without context makes navigation harder for screen readers; they should use a `<nav>` with descriptive `aria-label`s. Focus indicators are often stripped out when heavily styling tags with Tailwind.
**Action:** Always wrap tag lists in `<nav aria-label="Tags">`, apply consistent slugification to href links and their corresponding target ids, add the `scroll-mt-*` class to anchor targets to prevent the fixed header from covering them, and explicitly add `:focus-visible` styles (`focus-visible:ring-2 focus-visible:ring-primary...`) to ensure keyboard navigability.
