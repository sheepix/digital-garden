## 2024-05-15 - Deep Linking in Fixed Header Layouts
**Learning:** When using deep linking to jump to specific generated elements within a page that uses a fixed header layout, the target ID on the element will cause the browser to align the element with the absolute top of the viewport, which unfortunately places it directly beneath the fixed header, hiding it from view. Combining the ID attribute target with CSS scroll margins (e.g. `scroll-mt-24` in Tailwind) is a crucial accessibility necessity to ensure that targeted content is placed within the actual readable area of the screen upon navigation.
**Action:** When implementing anchor/deep links across dynamic component generations, ensure that the target wrapper container receives both an `id` that can be targeted safely and a scroll-margin class (`scroll-mt-*`) to offset the header overlap.
