## 2024-03-04 - Semantic Tag Navigation & Focus States
**Learning:** Using simple `<div>` elements for clusters of links (like a tag list) deprives screen reader users of structural context. Furthermore, custom styling on interactive elements must explicitly account for keyboard focus (`:focus-visible`), as browser defaults are often insufficient or stripped by CSS resets, and should be coordinated with dark mode themes.
**Action:** When implementing lists of links (like tags or breadcrumbs), always use semantic elements like `<nav>` with descriptive `aria-label`s. In addition, when building custom components with Tailwind, proactively include `focus-visible` utility classes (e.g., `focus-visible:ring-2 focus-visible:ring-primary focus-visible:outline-none`) and adjust offset colors for dark mode context (`dark:focus-visible:ring-offset-stone-900`) to ensure accessible keyboard navigation.
