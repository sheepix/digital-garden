## 2024-05-18 - Semantic Tags Component Navigation & Focus States
**Learning:** Tag components are often structured visually using `<div>` wrappers instead of semantic HTML, lacking correct ARIA roles for screen readers. Furthermore, because these are functional navigation elements rather than just badges, proper deep linking and explicit keyboard focus indicators (e.g., `focus-visible:ring-2`) are crucial for an accessible routing experience.
**Action:** Always wrap lists of tag links in `<nav aria-label="Page tags">` (or similar semantic containers), ensure the anchor dynamically deep-links via encoded slug formats instead of a generic route, and implement customized `focus-visible` styling (handling dark mode offsets appropriately) so keyboard-only users can clearly perceive the active element.
