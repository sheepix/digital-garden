## 2024-05-24 - Accessible Keyboard Navigation in Dark Mode
**Learning:** Using standard `focus-visible` outlines can cause contrast issues in dark mode contexts if the element background and outline color bleed together against a dark page background. Standard `ring-offset` values (which default to white) create a harsh, jarring visual break in dark mode that looks broken or unfinished.
**Action:** When implementing interactive elements, use Tailwind's `focus-visible` utility classes and explicitly coordinate offset colors for dark mode context (e.g., `dark:focus-visible:ring-offset-stone-900`). This ensures accessible keyboard navigation that maintains adequate contrast without breaking the visual design system of the dark theme. This is a reusable pattern for all focus states across the application.
